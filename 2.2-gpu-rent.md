# 2.2 Rental GPU di Vast.ai

Panduan ini mencakup cara menyewa server GPU di Vast.ai dan menghubungkannya untuk keperluan training model.

---

## 1. Teori: Mengapa Perlu Rental GPU?

### Masalah:
Training model Computer Vision modern (seperti YOLOv8/v11) membutuhkan daya komputasi paralel yang sangat besar. Menggunakan laptop pribadi seringkali tidak efisien karena:
1.  **Waktu Training Lama:** Bisa memakan waktu berhari-hari pada CPU atau GPU laptop biasa (GTX/RTX mobile).
2.  **Panas & Hardware:** Menyiksa hardware laptop dalam durasi lama berisiko *overheat*.
3.  **VRAM Terbatas:** Kartu grafis laptop biasanya memiliki VRAM kecil (4-6GB), membatasi ukuran *batch size* dan resolusi gambar yang bisa diproses.

### Solusi:
Solusinya adalah menyewa **Cloud GPU** melalui platform seperti **Vast.ai**.
1.  **Ekonomis:** Sistem *Peer-to-Peer* rental membuat harga sewa GPU *high-end* (seperti RTX 3090/4090) jauh lebih murah dibanding Cloud Provider besar (AWS/GCP).
2.  **Kinerja Tinggi:** Anda bisa menyewa GPU dengan VRAM besar (24GB+) untuk training yang jauh lebih cepat (hanya hitungan jam, bukan hari).
3.  **Fleksibel:** Bayar hanya saat Anda pakai (per jam).

## 2. Persiapan (Prerequisites)

### A. Akun Vast.ai

1. Buka [https://cloud.vast.ai](https://cloud.vast.ai/).
2. Login atau Buat akun jika belum memiliki.
3. Pastikan Anda sudah mengisi saldo (Credit) di menu **Billing** agar bisa menyewa instance.

### B. Generate & Registrasi SSH Key

Vast.ai memerlukan SSH Key (Public Key) untuk mengamankan akses ke server Anda. Langkah ini wajib dilakukan sebelum menyewa instance.

#### Langkah 1: Generate SSH Key (di Laptop Lokal)

Lakukan ini di terminal lokal Anda (PowerShell, CMD, atau Terminal Linux/WSL).

**Cek apakah key sudah ada:**
```bash
ls ~/.ssh/
```
Lihat apakah ada file berakhiran `.pub` (misalnya `id_rsa.pub` atau `id_ed25519.pub`).

*   **Jika file ada:** Lanjut ke Langkah 2 (Ambil Public Key).
*   **Jika tidak ada:** Buat baru dengan perintah berikut:

```bash
# Opsi 1: RSA (Standar)
ssh-keygen -t rsa

# Opsi 2: ED25519 (Lebih Modern & Aman)
ssh-keygen -t ed25519
```
*(Tekan Enter untuk semua prompt, biarkan password kosong agar akses lebih cepat).*

#### Langkah 2: Ambil Public Key

Tampilkan isi public key Anda di terminal:

```bash
# Sesuaikan nama file dengan yang Anda buat
cat ~/.ssh/id_rsa.pub
# Atau
cat ~/.ssh/id_ed25519.pub
```

Salin **seluruh teks** yang muncul (biasanya dimulai dengan `ssh-rsa ...` atau `ssh-ed25519 ...`).

## 3. Lab Hands-On

### Langkah 1: Registrasi SSH Key di Vast.ai

1. Kembali ke dashboard [Vast.ai](https://cloud.vast.ai/).
2. Masuk ke menu **Account** (ikon orang) -> **SSH Keys**.
   
   ![Keys Menu](../image/training/rent-gpu/image.png)

3. Klik **ADD SSH KEY** atau tombol serupa.
   
   ![New Key Button](../image/training/rent-gpu/image%201.png)

4. **Paste** key yang sudah disalin ke kolom yang tersedia, lalu klik **CREATE**.
   
   ![Paste Key Dialog](../image/training/rent-gpu/image%202.png)


### Langkah 2: Sewa Instance

1. Masuk ke menu **Client** -> **Search**.
2. **Filter GPU:** Cari yang sesuai kebutuhan dan budget (contoh: RTX 3060, RTX 3090, atau RTX 4090).
3. **Image Configuration (PENTING):**
   - Klik **Edit Image & Config** (atau "Select Template") di kiri atas setiap kartu GPU atau di bar filter.
   
   ![Select Template](../image/training/rent-gpu/image%203.png)

   - Pilih tab **Recommendation** atau **Docker Hub**.
   - Gunakan image `pytorch/pytorch` (sebagai base image yang stabil).
   
   ![pytorch/pytorch Selection](../image/training/rent-gpu/image%204.png)

   - Pilih **Launch Mode**: Pilih **Run interactive shell server, SSH, & Jupyter**.
   - Atur **Disk Space** (Gigabytes) sesuai kebutuhan dataset (disarankan minimal 30GB - 50GB).

4. Klik tombol biru **RENT** pada instance yang dipilih.


### Langkah 3: Akses Instance

Setelah menyewa, masuk ke menu **Instances** di sidebar kiri. Tunggu hingga status instance menjadi **Running** (tombol biru).

Ada 2 cara untuk mengakses instance Anda:

#### Cara 1: Menggunakan Jupyter Lab (GUI)

Cara ini paling mudah dan cocok untuk pemula atau debugging visual.

1. Di menu **Instances**, tunggu hingga tombol **OPEN** muncul (tombol biru).
   
   ![image.png](../image/training/rent-gpu/image%206.png)

2. Klik tombol **OPEN** tersebut.
3. Akan terbuka tab baru atau popup. Lihat bagian **Jupyter** (biasanya di kanan atas halaman perantara), lalu klik **Launch Application** (atau klik link Jupyter yang tersedia).
   *   *Note: Jika tidak terbuka, pastikan Pop-up Blocker di browser Anda dimatikan.*

   ![image.png](../image/training/rent-gpu/Screenshot_2026-01-20_102437.png)

#### Cara 2: Menggunakan SSH (Terminal)

Cara ini lebih stabil untuk training jangka panjang dan profesional.

1. Di menu **Instances**, lihat pada kartu instance Anda.
2. Klik tombol **Connect** (biasanya ikon terminal atau panah `>_`).
3. Salin perintah SSH yang ditampilkan. Formatnya mirip seperti ini:
   `ssh -p [PORT] root@[IP_ADDRESS] -L 8080:localhost:8080`
4. Buka terminal di laptop lokal Anda, lalu paste dan jalankan perintah tersebut.

**Contoh perintah:**
```bash
ssh -p 12345 root@1.2.3.4 -L 8080:localhost:8080
```

> **Catatan:** Flag `-L 8080:localhost:8080` digunakan jika Anda ingin meneruskan port (port forwarding), misalnya untuk melihat visualisasi pelatihan di browser lokal.
>
> Jika koneksi *Direct* (langsung) gagal/timeout, coba gunakan opsi *Proxy* yang disediakan Vast.ai (biasanya ada opsi switch di modal Connect).

## 4. Analisis & Verifikasi

Jika Anda telah melakukan langkah-langkah di atas dengan benar, berikut adalah hasil akhir yang seharusnya Anda dapatkan:

### 1. Akses Terminal Berhasil
Anda berhasil masuk ke terminal server (baik via SSH di terminal lokal atau Jupyter Terminal). Tanda prompt biasanya berubah menjadi `root@instance-id:~#`.

### 2. GPU Terdeteksi
Jalankan perintah ini di terminal server untuk memastikan driver NVIDIA berjalan normal:
```bash
nvidia-smi
```
*   **Expected Output:** Tabel yang menampilkan nama GPU (misal: NVIDIA GeForce RTX 3090), VRAM (Usage/Total), dan versi CUDA Driver.
*   **Jika Error:** Jika muncul pesan "command not found" atau error driver, coba sewa instance lain atau restart.

### 3. Koneksi Internet Server
Pastikan server bisa download file. Cek dengan ping:
```bash
ping google.com -c 3
```
