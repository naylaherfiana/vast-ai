Setelah berhasil menyewa dan mengakses instance (baik via Jupyter atau SSH), langkah selanjutnya adalah menyiapkan lingkungan (environment) dan menjalankan proses training.

Pilih metode yang sesuai dengan cara akses Anda sebelumnya.

> **Tips:** Sebelum memulai training, Anda bisa melihat **[2.6 Experimental Report](2.6-experimental-report.md)** untuk melihat tabel komparasi parameter (Batch Size, Model Size, Augmentasi) yang sudah kami uji coba sebelumnya dan menghasilkan performa terbaik.

## 1. Teori: Mengapa melakukan training model sendiri?

### Masalah:
Model bawaan (Pre-trained) seperti YOLOv8n/s/m.pt dilatih menggunakan dataset umum (COCO Dataset) yang berisi 80 kelas objek global (mobil, orang, jerapah, dll). 
-   **Objek Tidak Spesifik:** Model ini **tidak mengenali** objek spesifik kebutuhan industri, misalnya membedakan "Truk Logistik" vs "Truk Sampah", atau mendeteksi "Helm Proyek".
-   **Sudut Pandang Berbeda:** Model umum biasanya dilatih dengan foto *eye-level*. Jika digunakan pada CCTV (*high-angle*) atau Drone (*top-down*), akurasinya akan anjlok.

### Solusi:
Kita perlu melakukan proses **Training** (atau lebih tepatnya *Fine-Tuning*).
-   Kita mengambil "otak" model yang sudah pintar mengenali bentuk dasar (garis, kotak, lingkaran), lalu melatihnya ulang menggunakan **Dataset Spesifik** yang sudah kita siapkan.
-   Hasil akhirnya adalah file bobot baru (`best.pt`) yang memiliki keahlian khusus mengenali objek di lingkungan proyek Anda.

#### Langkah 2: Download Dataset & Training
Kita akan menggunakan script Python untuk mendownload dataset dan menjalankan training.

Copy kode berikut ke cell baru dan jalankan:

```python
from roboflow import Roboflow
from ultralytics import YOLO

# --- 1. Download Dataset ---
# Masukkan API Key Roboflow Anda
rf = Roboflow(api_key="YOUR_ROBOFLOW_API_KEY") 
dataset = rf.workspace("workspace-name").project("project-name").version(1).download("yolov8")

# --- 2. Mulai Training ---
model = YOLO('yolov8s.pt')

# Training dengan setting standar
results = model.train(
    data=f"{dataset.location}/data.yaml",
    epochs=100,
    imgsz=640
)
```
*Script di atas akan otomatis menjalankan training. Jika ingin pengaturan lebih lanjut (augmentasi, batch size), Anda bisa menambahkannya di dalam `model.train()`.*

### Langkah 3: Monitoring Proses (Real-time)
Lihat baris log yang berjalan.
*   **Box Loss:** Harus makin turun (mendekati 0.5 - 1.0).
*   **mAP50 (Mean Average Precision):** Harus makin naik (mendekati 1.0 atau 0.9).
*   *Analisis:* Jika Box Loss tidak turun-turun setelah 10 epoch, mungkin *Learning Rate* terlalu besar atau dataset rusak (label ngaco).

### Metode 2: SSH (Terminal)

Metode ini dilakukan melalui terminal (PowerShell/CMD/Terminal) yang sudah terhubung via SSH ke instance.

#### Langkah 1: Tips: Gunakan TMUX (Penting!)
Agar training tidak berhenti saat koneksi internet laptop putus, gunakan `tmux`.

```bash
# Membuat sesi baru bernama 'traffic'
tmux new -s traffic
```
*Jika koneksi putus, Anda bisa login lagi dan ketik `tmux attach -t traffic` untuk kembali ke layar proses.*

#### Langkah 2: Setup & Install
Install library ultralytics di server.

```bash
pip install ultralytics roboflow
```

#### Langkah 3: Download Dataset
Buat folder khusus dan download dataset Anda menggunakan curl (dapatkan command ini dari Roboflow -> Export -> YOLOv8 -> Show Download Code -> Terminal).

```bash
mkdir my_dataset
cd my_dataset

# Contoh format command (Ganti dengan punyamu!)
curl -L "https://universe.roboflow.com/ds/xyz...?key=YOUR_KEY" > roboflow.zip; unzip roboflow.zip; rm roboflow.zip
```
Pastikan di dalam folder tersebut sekarang ada `data.yaml`.

#### Langkah 4: Jalankan Training (CLI)
Kembali ke folder utama (atau jalankan dari mana saja asalkan path data benar).

```bash
yolo detect train \
  data=my_dataset/data.yaml \
  model=yolov8s.pt \
  epochs=100 \
  imgsz=640 \
  batch=-1 \
  project=traffic_project \
  name=speed_run_cli
```

#### Langkah 5: Monitoring Proses (Real-time)
Lihat baris log yang berjalan.
*   **Box Loss:** Harus makin turun (mendekati 0.5 - 1.0).
*   **mAP50 (Mean Average Precision):** Harus makin naik (mendekati 1.0 atau 0.9).
*   *Analisis:* Jika Box Loss tidak turun-turun setelah 10 epoch, mungkin *Learning Rate* terlalu besar atau dataset rusak (label ngaco).

#### Langkah 6: Apa yang diambil? (Download ke Lokal)
Setelah selesai, file hasil ada di server. Anda perlu mengambilnya menggunakan perintah `scp` (Secure Copy) dari **Terminal Laptop Lokal Anda** (bukan terminal SSH server).

Buka terminal baru di laptop, lalu jalankan:

```bash
# Format: scp -P [PORT] -i [KEY] root@[HOST]:[PATH_SERVER] [PATH_LOKAL]

# Contoh mengambil file best.pt:
scp -P 12345 -i ~/.ssh/id_ed25519 root@1.2.3.4:/root/traffic_project/speed_run_cli/weights/best.pt ./model_traffic_terbaik.pt
```

*   Ganti `12345` dan `1.2.3.4` sesuai info SSH Vast.ai Anda.
*   Path server biasanya `/root/...` jika Anda login sebagai root.

## 4. Analisis & Verifikasi

Jika Anda telah melakukan langkah-langkah di atas dengan benar, berikut adalah hasil akhir yang seharusnya Anda dapatkan:

### 1. Proses Training Berjalan
Anda melihat progress bar training bergerak (Epoch 1/100, 2/100, dst).
-   **Loss:** Nilai *box_loss*, *cls_loss*, dan *dfl_loss* cenderung turun seiring bertambahnya epoch.
-   **Metrics:** Nilai *mAP50* dan *mAP50-95* cenderung naik.

### 2. File Hasil Tersimpan
Setelah training selesai, pastikan Anda memiliki file-file berikut (baik di server maupun setelah di-download):
-   `best.pt`: Model hasil training terbaik. Ukurannya sekitar 22MB (untuk YOLOv8s) hingga ratusan MB (untuk model besar).
-   `results.csv`: Log performa per epoch.
-   `results.png`: Grafik visualisasi performa training.

### 3. Ukuran File Valid
Cek ukuran file `best.pt` yang sampai di laptop Anda. Jika ukurannya `0 bytes` atau sangat kecil (<1KB), berarti proses download (SCP) gagal.
